Reliable UDP Media Delivery Summary

Overview

The project implements a lightweight reliability layer over UDP by using controlled packet duplication. The goal is to reduce packet loss while maintaining bandwidth below the required 2× overhead limit and keeping playback latency as low as possible.

Sender Architecture

The sender continuously receives packets from the source application. Each packet is transmitted immediately to minimize latency. To improve reliability, approximately 90% of packets are duplicated. Duplication is skipped for packets whose sequence number is divisible by 10, keeping the average bandwidth overhead below the required limit.

  • Receive packet from source
  • Immediately transmit the original packet
  • Duplicate about 90% of packets
  • No buffering or packet encoding

Receiver Architecture

The receiver immediately forwards every newly received packet to the player. A fixed-size ring buffer stores recently observed sequence numbers to suppress duplicate packets. This avoids delivering multiple copies generated either by the sender or by the relay.

  • Receive packet
  • Check whether it has already been received
  • Discard duplicate packets
  • Immediately forward unique packets

Design Choices

  • Immediate forwarding to minimize playout delay.
  • Packet duplication instead of Forward Error Correction.
  • Duplicate suppression using sequence numbers.
  • Bandwidth maintained below the 2× assignment limit.
  • Simple implementation with low computational overhead.

Advantages

  • Very simple protocol.
  • Low processing overhead.
  • No recovery delay introduced by parity generation.
  • Good performance under moderate packet loss.

Limitations

  • Packets cannot be recovered if both the original and duplicate are lost.
  • Performance decreases under higher packet loss or large network delays.
  • No retransmission or Forward Error Correction is used.

Final Design

The final implementation favors simplicity and low latency by combining immediate packet forwarding with controlled packet duplication and duplicate suppression. This approach improves reliability while keeping the bandwidth overhead safely below the required limit.
